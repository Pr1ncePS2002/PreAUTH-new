## PHASE 2 — THE FIELD MAPPING PROBLEM (Critical — Solve This First)

### The Problem
After Document AI processes a document, it returns key-value pairs like:
```json
{
  "Policy Holder Name": "Ramesh Kumar",
  "Date of Birth": "12/04/1978",
  "Sum Insured": "500000"
}
```

But our coordinate JSON schema (e.g., `analyzed/Ericson TPA Preauth.json`) has field IDs like:
```json
{
  "field_id": "insured_name",
  "field_id": "dob",
  "field_id": "sum_insured_amount"
}
```

These will NEVER match directly. We need a mapping layer.

### Solution — Build a `field_mapping.json` + Mapping Engine

#### Structure of `field_mapping.json`
```json
{
  "insured_name": {
    "aliases": ["Policy Holder Name", "Name of Insured", "Patient Name", "Insured Name"],
    "document_types": ["policy_card", "aadhaar", "preauth_form"],
    "gemini_description": "Full name of the insured/policy holder"
  },
  "dob": {
    "aliases": ["Date of Birth", "DOB", "Birth Date", "D.O.B"],
    "document_types": ["aadhaar", "pan", "policy_card"],
    "gemini_description": "Date of birth in any format"
  }
}
```

#### Mapping Engine Logic (implement in `services/mapping_engine.py`)
```python
def map_docai_to_schema(docai_output: dict, schema_fields: list, mapping: dict) -> dict:
    """
    1. Try exact match on field_id
    2. Try alias match from field_mapping.json
    3. Try fuzzy string match (rapidfuzz)
    4. If still unmatched, call Gemini to suggest mapping
    5. Log all unmatched fields for manual review
    """
```

#### Auto-Generate Mappings
When processing a new form for the first time:
- Pass the Document AI output keys + the JSON schema field IDs to Gemini
- Ask Gemini to suggest the mapping as JSON
- Store the confirmed mapping in `field_mapping.json`
- Never regenerate confirmed mappings
