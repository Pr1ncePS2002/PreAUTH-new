# Copilot Context — TPA Pre-Authorization Form Automation

This repo is a FastAPI backend + single-page frontend that:
1) accepts a staff-entered **MRD number** and multiple patient documents (Aadhaar, policy card, estimates, clinical notes, etc.),
2) runs OCR + key-value extraction using **Gemini Vision**,
3) **validates MRD** against extracted document data (cross-verification),
4) auto-detects the most likely TPA form from insurance company name,
5) maps extracted keys into a **coordinate-based form schema** (JSON),
6) lets staff review/edit the mapped fields in a **two-phase, tabbed UI**,
7) handles the cost section specially — writes **"ESTIMATE ATTACHED"** on the PDF and only populates the **sum total** amount,
8) detects **GIPSA** cases and generates a **PPN Declaration** form,
9) generates a filled PDF + merges PPN + uploaded attachments into a **single claim package** named by MRD number.

## Quick start (local)

1) Create a `.env` in repo root:

```env
GEMINI_API_KEY=...
GEMINI_MODEL=gemini-2.5-flash
JWT_SECRET=dev-secret-change-in-production
```

2) Install deps:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

3) Run the app:

```powershell
.\venv\Scripts\uvicorn.exe app:app --reload --port 8001
```

4) Open the UI: http://localhost:8001/ui

## Project layout

- `app.py` — Main FastAPI app (~1370 lines). Workflow endpoints, session persistence, TPA detection, GIPSA/PPN orchestration, MRD validation, PDF merge.
- `frontend/index.html` — Two-phase single-page UI (~1470 lines). Phase 1: MRD + upload + OCR. Phase 2: 6-tab form (Patient, Insurance, Clinical, Cost, Attachments, Generate).
- `services/ocr_service.py` — OCR extraction via Gemini Vision.
- `services/mapping_engine.py` — Maps OCR keys to schema field IDs (exact → alias → fuzzy → Gemini fallback).
- `services/form_engine.py` — PDF overlay filling (ReportLab + PyPDF2). Renders "ESTIMATE ATTACHED" in bold.
- `services/pdf/generate_ppn_pdf.py` — PPN Declaration generator for GIPSA cases (50+ field aliases across 3 schemas).
- `services/pdf/merge_documents.py` — Merges TPA form + PPN + attachments into single PDF.
- `services/his_service.py` — Stub HIS integration (mock data).
- `config/field_mapping.json` — Canonical field IDs + alias lists (~80+ fields, 1770 lines).
- `templates/` — Blank TPA PDF templates.
- `analyzed/` — Coordinate schemas generated by Gemini/pdfplumber analysis.

## User flow

### Phase 1 — MRD + Upload + OCR
- Staff enters **MRD number** (required) → uploads documents → clicks **Extract Data**.
- Backend: saves files, runs parallel OCR, merges results, validates MRD against documents, auto-detects TPA, maps fields.
- MRD validation: verified (match) / mismatch (warning) / not_found_in_docs (info).

### Phase 2 — 6-Tab Form
- **Patient Details**: Hospital/provider + patient demographics.
- **Insurance & TPA**: Policy, insurance company, GIPSA toggle, PPN checkbox.
- **Clinical Details**: Doctor, diagnosis, hospitalization, injury.
- **Cost & Declarations**: "ESTIMATE ATTACHED" banner — only sum total editable. Individual line items hidden. On PDF generation, first cost field gets "ESTIMATE ATTACHED" text, only total gets the amount.
- **Attachments**: Additional supporting documents.
- **Generate PDF**: Checklist + summary card, Generate + Go Back card. Preview + download after generation.

### Claim Package
- Named `claim_package_MRD_{mrd_number}.pdf` (falls back to `claim_package_{form_id}.pdf`).
- Contains: filled TPA form + PPN declaration (if GIPSA) + all uploaded attachments.

## Cost section behavior

Individual cost fields (room rent, ICU, OT, etc.) are NOT populated with amounts:
- First cost line-item field → "ESTIMATE ATTACHED" (bold on PDF).
- Sum total field → actual amount from estimate.
- Frontend shows banner + only sum total input.

## Key technical details

- **Coordinate system**: Schema `x,y` is top-left origin. Fill engine converts: `y_pdf = page_height - y_schema`.
- **Mapping**: MappingEngine (alias+fuzzy, threshold 70) → schema-label matching (threshold 65) → hospital injection → age calculation.
- **GIPSA**: 26 known TPA names. Auto-detected from insurance company. PPN generator uses merged raw_ocr + mapped_data.
- **Sessions**: JSON files in `sessions/` — survive server restarts.
